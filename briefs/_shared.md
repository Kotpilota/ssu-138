## 8. Технические требования

- **Стек:** HTML5 + CSS3 + ванильный JS. Без фреймворков, без сборщиков, без npm.
- **Файлы:** `index.html`, `style.css`, `script.js`. Можно разбить CSS на `reset.css` + `style.css` если оправданно.
- **Шрифты:** через Google Fonts с `display=swap`.
- **Иконки:** inline SVG или [Lucide](https://lucide.dev/) через CDN.
- **Адаптив обязателен:** breakpoints 1440 / 1024 / 768 / 375. Mobile-first или desktop-first — на усмотрение, главное чтобы работало.
- **Плейсхолдеры картинок:** `https://placehold.co/{w}x{h}/{bg}/{fg}?text={подпись}` — для каждого объекта подписать кириллицей через `encodeURIComponent`. Цвета `bg`/`fg` подобрать под палитру варианта.
- **Логотип:** скопировать `icon.svg` в папку варианта.
- **Форма:** `onsubmit` → `preventDefault` → `alert('Заявка отправлена. Мы свяжемся с вами в течение рабочего дня.')` + `reset()`.
- **Карта:** статичный плейсхолдер-блок (можно `<iframe>` Яндекс.Карт с координатами Москвы или просто div с фоном-плейсхолдером и подписью).
- **Производительность:** критический CSS inline в `<head>` приветствуется, не обязателен.
- **Доступность:** alt-тексты у всех картинок, semantic HTML (`<header>`, `<main>`, `<section>`, `<article>`, `<footer>`), focus-visible на интерактивных элементах.
