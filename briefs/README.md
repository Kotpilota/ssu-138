# Брифы для агентов Sonnet 4.6

Здесь лежат 4 концепции дизайна сайта ССУ-138. Каждый агент получает 2 файла на вход:

1. `_shared.md` — общий контекст: компания, факты, структура страницы, технические требования
2. `variant-X.md` — визуальная концепция конкретного варианта

## Соответствие вариантов

| Вариант | Концепция | Куда писать |
|---|---|---|
| **A** | Премиум-монументальный (госкорпорация) | `variants/variant-A/` |
| **B** | Industrial / Engineering (linear-style) | `variants/variant-B/` |
| **C** | Swiss-минимализм (фото-ориентированный) | `variants/variant-C/` |
| **D** | Конверсионный B2B-лендинг с калькулятором | `variants/variant-D/` |

## Шаблон промпта для агента

```
Ты — фронтенд-разработчик. Задача: сделать одностраничный сайт-лендинг для строительной компании ССУ-138.

Прочитай два файла в этом репозитории:
1. briefs/_shared.md — факты о компании и структура страницы (обязательно)
2. briefs/variant-A.md — визуальная концепция твоего варианта (замени A на нужную букву)

Также есть icon.svg в корне — это логотип компании.

Сделай статический сайт в папке variants/variant-A/ (соответственно):
- index.html, style.css, script.js
- Скопируй icon.svg в эту папку
- Используй placehold.co для всех картинок (см. _shared.md)
- Без фреймворков, без npm, без сборщиков
- Адаптив 1440 / 1024 / 768 / 375
- Все интерактивные элементы (формы, табы, калькулятор если есть) должны работать на ванильном JS

После завершения открой index.html и проверь:
- Все секции из структуры _shared.md на месте
- Адаптив работает
- Формы валидируются и показывают alert при отправке
- Никаких console.error в консоли

Не делегируй работу другим агентам. Не задавай вопросов — действуй по брифам, в спорных моментах выбирай вариант, который ближе к концепции.
```

## Запуск 4 агентов параллельно

Все 4 агента нужно запустить **одним сообщением** с 4 параллельными вызовами Task tool, чтобы они работали одновременно. Каждому подставить свою букву варианта.

## После сборки

1. Открыть все 4 `index.html` в браузере
2. Сравнить визуально и по конверсионным точкам
3. Выбрать направление (или гибрид элементов из разных)
4. Дальше — реальные фото, тексты, бэкенд для форм, SEO
